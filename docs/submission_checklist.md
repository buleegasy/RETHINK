# 参赛提交核对清单

本文用于最终提交前逐项检查材料，避免遗漏文件、格式或验证步骤。

## 1. 提交材料

### 1.1 作品设计报告

- [ ] 文件为 PDF 格式。
- [ ] 报告包含背景、目的、功能、创新点、支撑数据。
- [ ] 报告紧贴问卷调研与开放评论。
- [ ] 报告写清从问题发现到原型实现的逻辑。
- [ ] 报告包含复杂逻辑图，而不是只有简单截图。
- [ ] 报告附录包含 RAG 外部参考文献与网页来源完整清单。
- [ ] 报告查重率不超过 30%。

当前相关文件：

- `reports/作品设计报告.md`
- `reports/青少年心理支持智能体作品设计报告.docx`
- `reports/figures/`

### 1.2 过程性证明材料

- [ ] 包含创作全流程步骤图片。
- [ ] 包含参赛者成果介绍视频链接。
- [ ] 视频链接可以直接在线播放。
- [ ] 如有播放密码，已在正文中备注。
- [ ] 正文材料以图片为主。
- [ ] 最终文件为 PDF 格式。

建议截图：

- 问卷页面。
- 问卷后台统计。
- 表情包入口。
- 普通对话。
- 学业压力 RAG 后台推演。
- 校园欺凌 forced_safety 后台推演。
- 危机路径 Crisis_Escalation。
- 终端 smoke test 通过结果。

### 1.3 查重报告

- [ ] 使用知网、维普、万方等权威系统检测。
- [ ] 查重率小于或等于 30%。
- [ ] 查重报告与作品设计报告对应版本一致。
- [ ] 查重报告文件为 PDF。

## 2. 文件格式

- [ ] 所有成果文件均为 PDF。
- [ ] 单个文件不超过 50MB。
- [ ] 视频以链接形式插入，不直接嵌入大文件。
- [ ] 文件命名清晰，包含项目名或材料类型。

推荐文件名：

```text
RE-THINK_Agent_作品设计报告.pdf
RE-THINK_Agent_过程性证明材料.pdf
RE-THINK_Agent_查重报告.pdf
```

## 3. 系统验证

提交前建议执行：

```bash
./node_modules/.bin/tsc -p worker/tsconfig.json --noEmit
./node_modules/.bin/tsc -p web/tsconfig.json --noEmit
npm run test:agent
```

测试通过后，记录以下内容到过程性证明材料：

- 测试时间。
- 测试命令。
- 四类场景均为 PASS。
- Worker 版本号，如部署输出中的 `Current Version ID`。

- [ ] 问卷可访问：https://survey.rethink.buleegasy.space

建议视频结构：

1. 项目一句话介绍。
2. 问卷调研发现。
3. 表情包低压力入口。
4. 学业压力案例。
5. 后台推演面板。
6. 校园欺凌 forced_safety 案例。
7. 危机路径案例。
8. 自动化测试与总结。

## 6. 最终提交前人工检查

- [ ] 报告中不直接写评分标准。
- [ ] 报告中引用问卷原话时已保持原文，不歪曲含义。
- [ ] 报告中没有把 AI 说成可替代医生。
- [ ] 危机干预表述强调现实求助和专业资源。
- [ ] 隐私边界表述清楚。
- [ ] RAG 文献来源完整。
- [ ] 所有链接可以打开。
- [ ] PDF 在不同设备上打开版式不乱。
